=> Pagination: spliting large data into smaller pages
    * server-side: server decides what data to send per page (?page=2) (recommended)
        server runs this query in db (SELECT * FROM users LIMIT 10 OFFSET 10;)
        flask-example: 
            page = request.args.get('page', 1, type=int)
            per_page = 10
            users = User.query.paginate(page=page, per_page=per_page)

    * client-side: server sends all data at once - browser splits pages using JS
        i.e(server sends 1000 items - JS shows only 10 per page)

=> Pagination Methods: 
    * Normal (Page + offset)
        URL: (/users?page=3)
        DB: (LIMIT 10 OFFSET 20)

    * Cursor Based (Modern + Fast)                      (_______BITLY_CLONE_______)
        instead of page numbers, use a cursor (last item id or timestamp)
            URL: (/posts?after=105)
            DB: SELECT * FROM posts WHERE id >105 LIMIT 10;

        Flask-example: 
            last_id = request.args.get('after', 0, type=int)
            posts = Post.query.filter(Post.id > last_id)\
                    .limit(10)\
                    .all()

==> RBAC and ABAC
Access Control: Who can access what? 

RBAC: Role based access control 
    (access depends on the user's ROLE)
    like (admin, editor, user, manager)
    
    Flask-example: 
        User Model: 
            class User(db.Model):
                id = db.Column(db.Integer, primary_key=True)
                role = db.Column(db.String(20))
        from flask_login import current_user
        @bp.route('/admin')
        def admin_panel():
            if current_user.role != 'admin'
                abort(403)
            return render_template('admin.html')

ABAC: Attribute based access control
    (access depends upon Attributes (data conditions))
        like (user.department, user.age, resouce.owner_id, time of day, location)
    banking, 18+ content etc. 

    Flask-example: 
        @bp.route('/edit/<int:post_id>')
        def edit_post(post_id):
            post = Post.query.get_or_404(post_id)
            if post.author_id != current_user.id:
                abort(403)
            return render_template('edit.html')
